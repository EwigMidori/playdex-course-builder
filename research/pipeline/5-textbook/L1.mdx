# 算法交易基础与金融市场

## 为什么这一课重要 {#why-this-lesson-matters}

想象一下，你不需要一直盯着屏幕，电脑就能帮你做交易决策并自动下单。这就是算法交易的核心魅力。

在当今金融市场，算法交易已经不再是边缘玩法。在美国股市，超过60%的交易量由算法完成；在欧洲，这个比例是40%；在外汇市场，也有25%的交易是算法驱动的。整个市场预计以每年12.7%的速度增长，到2028年将达到320亿美元。

这门课将带你系统性地理解算法交易的基础知识、市场机制和关键技术。无论你未来是从事量化投资、金融科技还是传统金融，这些知识都是不可或缺的基石。

<KeyPoint>
算法交易就是用数学模型和计算机程序来生成交易信号并自动执行交易。它的目标是最大化利润、控制交易成本、对冲和管理投资风险。
</KeyPoint>

## 课程全景与关键问题 {#concept-map}

### 什么是算法交易？

<Definition title="算法交易 (Algorithmic Trading)">
使用数学模型和计算机程序来生成交易信号（决策）并自动执行交易（下单）的方式。它也被称为 Algo Trading、Quant Trading、Robo-Trading 或 Program Trading。
</Definition>

算法交易主要做两件事：
1. **生成交易信号**：根据数学模型做出买入或卖出的决策
2. **自动执行**：将交易指令自动发送到市场并完成买卖

它的目标有三个：
- **最大化利润**：抓住更多交易机会
- **控制执行成本**：减少滑点、佣金等交易成本
- **对冲和管理投资风险**：通过算法自动控制风险敞口

<Example title="算法交易的日常工作">
假设你编写了一个算法，当某股票价格跌破其20日均线时自动买入，当价格涨回均线以上时自动卖出。这个算法会：
1. 持续监控市场价格和均线数据（数据输入）
2. 当条件满足时生成买入或卖出信号（决策）
3. 自动向交易所发送订单（执行）
整个过程无需人工干预。
</Example>

## 算法交易的市场趋势 {#market-trends}

### 市场渗透率

算法交易策略在不同市场的渗透率差异显著：

| 市场 | 算法交易占比 |
|------|------------|
| 美国股市 | 约60% |
| 欧洲股市 | 约40% |
| 外汇市场 | 约25% |
| 美国期权市场 | 约20% |

### 增长趋势

- 预计年复合增长率（CAGR）为12.7%
- 到2028年市场规模将达到320亿美元
- 北美是最大市场，亚太地区增长最快

### 机器人交易 vs 机器人投顾

<Definition title="机器人交易 (Robo-Trading)">
完全自动化的算法交易，包括决策和执行。系统自己判断买卖时机并自动下单。
</Definition>

<Definition title="机器人投顾 (Robo-Advisory)">
半自动化的投资建议服务，提供股票建议、市场总结和风险提醒，但不自动执行交易。
</Definition>

| 特征 | 机器人交易 | 机器人投顾 |
|------|-----------|-----------|
| 决策能力 | ✓ | ✓ |
| 执行能力 | ✓ | × |
| 自动化程度 | 全自动 | 半自动 |
| 输出 | 执行交易信号 | 股票建议、市场总结、风险提醒 |

<KeyPoint>
机器人交易是全自动的，自己做决策并执行交易；机器人投顾是半自动的，提供建议但不下单。
</KeyPoint>

## 交易机制与订单簿 {#trading-mechanism}

### 订单簿

<Definition title="订单簿 (Order Book)">
实时更新的买卖订单列表，按价格排序。它是市场撮合交易的核心工具。
</Definition>

订单簿包含两方：
- **买方队列（Bid Book）**：列出所有愿意买入的订单
- **卖方队列（Ask Book）**：列出所有愿意卖出的订单

### 买入价与卖出价

<Definition title="买入价 (Bid Price)">
买方愿意支付的最高价格。
</Definition>

<Definition title="卖出价 (Ask Price)">
卖方愿意接受的最低价格。
</Definition>

### 买卖价差

<Definition title="买卖价差 (Bid-Ask Spread)">
卖出价减去买入价的差额。价差越小，说明市场流动性越好，交易成本越低。
</Definition>

<Example title="买卖价差计算">
某股票的买入价为24元，卖出价为25元。
- 买卖价差 = 25 - 24 = 1元
- 百分比价差 = 2 × (25 - 24) / (25 + 24) × 100% ≈ 4.08%
</Example>

### 订单类型

#### 市价单

<Definition title="市价单 (Market Order)">
按当前最优价格立即成交的订单。保证成交，但不保证价格。
</Definition>

**适用场景**：当你需要立即执行交易，对价格不太敏感时。

#### 限价单

<Definition title="限价单 (Limit Order)">
指定一个价格，只有达到或更优时才成交的订单。保证价格，但不保证成交。
</Definition>

**适用场景**：当你希望以特定价格或更好价格买入/卖出时。

#### 止损单

<Definition title="止损单 (Stop Order)">
当价格触及设定水平后，自动变成市价单的订单。常用于止损或突破买入。
</Definition>

**适用场景**：限制亏损或捕捉突破行情。

<Example title="三种订单对比">
假设当前股票价格为100元：
- **市价单买入**：立即以当前最优价格（如100.10元）成交
- **限价单买入**：设定限价98元，只有价格跌到98元或以下才成交
- **止损单买入**：设定止损价105元，当价格涨到105元时自动变成市价单买入
</Example>

### 滑点

<Definition title="滑点 (Slippage)">
预期成交价与实际成交价之间的差距。
</Definition>

<Example title="滑点示例">
你设置算法在股价跌至50元时买入100股，但由于市场波动，实际成交价为51元。
- 滑点 = 51 - 50 = 1元/股
- 总滑点成本 = 100股 × 1元 = 100元
</Example>

<KeyPoint>
滑点通常发生在市场波动剧烈或流动性不足时。使用市价单时更容易产生滑点。
</KeyPoint>

## 做多、做空与盈亏计算 {#positions-pnl}

### 多头头寸

<Definition title="多头头寸 (Long Position)">
先买入资产，预期价格上涨后卖出获利。
</Definition>

**操作步骤**：买入 → 等待价格上涨 → 卖出

**最大风险**：损失全部本金（股价跌至0）

### 空头头寸

<Definition title="空头头寸 (Short Position)">
先借入资产卖出，预期价格下跌后买回获利。
</Definition>

**操作步骤**：借入股票 → 卖出 → 等待价格下跌 → 买回 → 归还股票

**最大风险**：理论亏损无上限（股价可以无限上涨）

<KeyPoint>
做空的风险比做多大得多，因为做多最大亏损有限（本金），而做空的理论亏损没有上限。
</KeyPoint>

### 盈亏公式

#### 做多盈亏公式

<Formula latex="PnL = Quantity \times (P_{current} - P_{entry})">
做多盈亏 = 数量 × (当前价 - 买入价)
</Formula>

#### 做空盈亏公式

<Formula latex="PnL = -1 \times Quantity \times (P_{current} - P_{entry})">
做空盈亏 = -1 × 数量 × (当前价 - 卖出价)
</Formula>

### 逐日盯市

<Definition title="逐日盯市 (Mark-to-Market)">
按当前市场价格计算资产或头寸价值的会计方法。它反映的是未实现盈亏（浮动盈亏）。
</Definition>

<Example title="盈亏计算示例">
**场景1：做多**
以每股20元买入300股股票A，当前股价25元。
- 盈亏 = 300 × (25 - 20) = 1500元（盈利）

**场景2：做空**
以每股50元卖空200股股票B，当前股价45元。
- 盈亏 = -1 × 200 × (45 - 50) = 1000元（盈利）

**场景3：做空亏损**
以每股40元卖空150股股票C，当前股价45元。
- 盈亏 = -1 × 150 × (45 - 40) = -750元（亏损）
</Example>

## 杠杆、保证金与交易系统 {#leverage-margin}

### 杠杆

<Definition title="杠杆 (Leverage)">
使用借入资金来放大投资回报的方式。
</Definition>

<Formula latex="Leverage\ Ratio = \frac{Asset}{Equity}">
杠杆比率 = 资产总值 / 自有资金
</Formula>

### 保证金

<Definition title="保证金 (Margin)">
投资者在经纪商处持有的自有资金，用于担保借款。它是创造杠杆的基础。
</Definition>

<Example title="杠杆计算示例">
你支付60万元购买价值100万元的股票：
- 杠杆比率 = 100万 / 60万 ≈ 1.67倍
- 如果股票上涨10%，你的自有资金回报率 = 1.67 × 10% = 16.7%
- 如果股票下跌10%，你的自有资金亏损率 = 1.67 × 10% = 16.7%
</Example>

### 购买力

<Definition title="购买力 (Buying Power)">
投资者可用于购买证券的最大资金量。
</Definition>

<Formula latex="Buying\ Power = Leverage\ Ratio \times Investor's\ Equity">
购买力 = 杠杆比率 × 自有资金
</Formula>

<Example title="购买力计算">
你有10万元本金，使用20倍杠杆：
- 购买力 = 20 × 10万 = 200万元
- 这意味着你最多可以购买200万元的证券
</Example>

### 交易系统类型

#### 基于订单的系统

<Definition title="基于订单的系统 (Order Based System)">
每笔交易独立管理，不支持部分平仓。
</Definition>

**典型平台**：MetaTrader、TradingView、MetaStock

#### 基于头寸的系统

<Definition title="基于头寸的系统 (Position Based System)">
交易按净头寸管理，支持部分平仓。
</Definition>

**典型平台**：Interactive Brokers、Binance、大部分银行系统

<Example title="两种系统的区别">
假设你开了3笔交易：
1. 买入2股ABC，价格123元
2. 卖出1股ABC，价格124元
3. 卖出1股ABC，价格125元

**基于订单的系统**：
- 每笔交易独立计算盈亏
- 不能只平掉其中一笔交易
- 必须同时关闭所有3笔交易才能实现盈亏

**基于头寸的系统**：
- 交易按净头寸管理（净多头0股）
- 支持部分平仓
- 盈亏可以立即实现

## 实施算法交易的基础设施 {#infrastructure}

### 算法是车，数据是燃料

<KeyPoint>
没有数据，算法寸步难行。数据是算法交易的输入和决策依据。
</KeyPoint>

### 数据工作的四个环节

| 环节 | 主要挑战 |
|------|---------|
| **采集** | 数据来源和清洗非常耗时费力 |
| **存储** | 海量非结构化数据（数值、文本、图像、音频、视频） |
| **管理** | 数据格式不一致，难以通过传统数据库保护和管理 |
| **处理** | 可行性和效率是关键问题 |

### 其他基础设施因素

除了数据工作，实施算法交易还需要考虑：

- **实时数据源**：获取实时市场行情
- **历史数据**：用于回测和策略开发
- **计算硬件**：运行算法和处理数据
- **交易所/经纪商连接**：与市场对接
- **网络和延迟**：低延迟对高频交易至关重要
- **订单路由系统**：将订单发送到最优交易场所
- **订单管理系统（OMS）**：管理、发送和跟踪订单
- **算法维护**：持续监控和优化算法
- **网络安全**：保护系统和数据安全

<Example title="基础设施的重要性">
假设你开发了一个优秀的交易算法，但如果：
- 数据源延迟5秒，你可能会错过最佳交易时机
- 网络延迟高，你的订单可能以更差的价格成交
- 没有订单管理系统，你可能无法有效跟踪订单状态
- 网络安全薄弱，你的算法可能被黑客攻击

因此，搭建完整的算法交易系统远不止写代码那么简单。
</Example>

## 易错点 {#pitfalls}

### 1. 混淆做空与做多的风险

<Pitfall>
**错误理解**：做空和做多的风险是一样的。
**正确理解**：做多最大亏损有限（本金），做空理论亏损无上限。股价可以涨到远高于做空价格，导致无限亏损。
</Pitfall>

### 2. 忽视滑点的影响

<Pitfall>
**错误理解**：滑点只是小问题，不会对交易结果产生重大影响。
**正确理解**：在高频交易或大额交易中，滑点可能成为最大的交易成本之一。尤其是在市场波动剧烈或流动性不足时，滑点可能远超预期。
</Pitfall>

### 3. 误用杠杆

<Pitfall>
**错误理解**：杠杆只放大收益，不放大风险。
**正确理解**：杠杆同时放大收益和风险。使用高杠杆时，小幅价格波动就可能导致本金全部亏完（爆仓）。
</Pitfall>

### 4. 混淆基于订单和基于头寸的系统

<Pitfall>
**错误理解**：所有交易平台都支持部分平仓。
**正确理解**：基于订单的系统（如MetaTrader）不支持部分平仓，必须关闭整个订单。基于头寸的系统（如Interactive Brokers）才支持部分平仓。
</Pitfall>

### 5. 低估基础设施的重要性

<Pitfall>
**错误理解**：只要算法好，基础设施不重要。
**正确理解**：算法是车，数据是燃料。没有可靠的数据源、低延迟的网络、稳定的订单管理系统，再好的算法也无法在真实市场中有效运行。
</Pitfall>

## 复习与自测 {#review-path}

### 核心概念回顾

1. **算法交易**：用数学模型和程序生成交易信号并自动执行
2. **市场趋势**：美国股市60%交易量由算法完成，市场CAGR 12.7%
3. **订单簿**：实时更新的买卖订单列表
4. **订单类型**：市价单（保证成交）、限价单（保证价格）、止损单（触发后变市价单）
5. **做多与做空**：做多先买后卖，做空先借后卖
6. **盈亏公式**：做多 PnL = Q × (P_current - P_entry)；做空 PnL = -1 × Q × (P_current - P_entry)
7. **杠杆与保证金**：杠杆放大收益和风险，保证金是自有资金
8. **交易系统**：基于订单（不支持部分平仓）vs 基于头寸（支持部分平仓）
9. **基础设施**：数据是燃料，需要采集、存储、管理、处理
