# 数据抓取与数据库管理

## 为什么这一课重要 {#why-this-lesson-matters}

想象一下，你正在监控 500 家公司的股价，手动刷新网页？太慢了。你需要一个自动化的方式，从网站上提取数据，这就是 <Term id="web_scraping" en="Web Scraping">网页抓取</Term>。

在量化交易中，数据是策略的基石。无论是获取实时行情、分析市场情绪，还是回测历史策略，都离不开高效的数据获取与管理。本课将带你掌握从网页抓取数据到存储管理的完整流程，为后续的策略开发打下坚实基础。

<KeyPoint>
本课的核心目标是让你能够：1) 理解网页结构；2) 使用 Python 工具抓取数据；3) 理解调整收盘价；4) 将数据存入数据库并设计合理的存储方案。
</KeyPoint>

## 课程全景与关键问题 {#concept-map}

本课围绕一个核心问题展开：**如何从互联网获取金融数据，并将其高效、准确地存储起来，用于后续的策略分析与回测？**

整个流程可以概括为：

1.  **理解数据来源**：网页由 <Term id="html" en="HyperText Markup Language">HTML</Term>、<Term id="css" en="Cascading Style Sheets">CSS</Term> 和 <Term id="javascript" en="JavaScript">JavaScript</Term> 构成。抓取的核心是解析 HTML 结构。
2.  **选择抓取工具**：根据网页的静态或动态特性，选择合适的 Python 库，如 <Term id="beautifulsoup" en="BeautifulSoup">BeautifulSoup</Term>、<Term id="lxml" en="lxml">lxml</Term>、<Term id="selenium" en="Selenium">Selenium</Term> 或专门的 <Term id="yfinance" en="yfinance">yfinance</Term>。
3.  **理解数据含义**：金融数据中的 <Term id="adjusted_close" en="Adjusted Close">调整收盘价</Term> 消除了公司事件的影响，是进行长期分析和回测的关键。
4.  **设计存储方案**：使用 <Term id="sqlite" en="SQLite">SQLite</Term> 等数据库持久化数据，并根据查询需求（如回测 vs 分析）设计合理的表结构。

<Checkpoint title="全局理解">
在深入学习之前，请思考：为什么不能直接使用从网页上看到的原始价格进行回测？为什么不能把所有数据都放在一个 Excel 文件里？
</Checkpoint>

## 网页的三大基石：HTML、CSS、JavaScript {#web-structure}

一个网页通常由三种技术共同构建：

- **<Term id="html" en="HyperText Markup Language">HTML</Term>**：定义网页的**结构**和**内容**，如标题、段落、表格、链接等。它是一种标记语言，不是编程语言。
- **<Term id="css" en="Cascading Style Sheets">CSS</Term>**：控制网页的**视觉呈现**和**布局**，如颜色、字体、间距、位置等。
- **<Term id="javascript" en="JavaScript">JavaScript</Term>**：为网页添加**交互**和**动态行为**，如点击弹窗、动态加载数据、表单验证等。

<KeyPoint>
网页抓取的核心是**解析 HTML 结构**来提取数据。CSS 和 JavaScript 主要影响呈现和交互，数据通常嵌入在 HTML 标签中。
</KeyPoint>

## 网页抓取工具与方法 {#scraping-tools}

Python 提供了多种库来应对不同的抓取场景。

### 1. BeautifulSoup：静态页面的利器

<Term id="beautifulsoup" en="BeautifulSoup">BeautifulSoup</Term> 是最流行的 Python 库之一，用于解析 HTML 和 XML 数据。它能将杂乱的 HTML 文档转换成一个可搜索的树形结构。

**基本流程：**
1.  使用 `requests` 库下载网页源码。
2.  使用 `BeautifulSoup` 解析源码。
3.  使用 `find()`、`find_all()` 等方法定位并提取数据。

<Example title="使用 BeautifulSoup 提取数据">
```python
import requests
from bs4 import BeautifulSoup

# 1. 下载网页
url = 'https://example.com'
response = requests.get(url)
text = response.text

# 2. 解析 HTML
soup = BeautifulSoup(text, 'html.parser')

# 3. 提取数据
# 提取第一个 <p> 标签
first_paragraph = soup.find('p')
print(first_paragraph.text)

# 提取所有 <a> 标签
all_links = soup.find_all('a')
for link in all_links:
    print(link.text, link.get('href'))
```
</Example>

<QuestionRef id="q_flash_bs4_methods_v1" />

### 2. lxml 与 XPath：精准定位

<Term id="lxml" en="lxml">lxml</Term> 是一个高效的 XML 和 HTML 解析库。它常与 <Term id="xpath" en="XPath">XPath</Term> 配合使用。XPath 是一种在 XML 文档中导航和选择节点的查询语言，可以像路径一样直接定位到深层元素。

**常见 XPath 语法：**
- `//element`：选择文档中所有名为 `element` 的元素。
- `//element[@attribute='value']`：选择属性 `attribute` 值为 `value` 的所有 `element` 元素。
- `//element[position()=1]`：选择第一个 `element` 元素。

<Example title="使用 XPath 定位表格">
```python
from lxml import html
import requests

url = 'https://en.wikipedia.org/wiki/List_of_S%26P_500_companies'
headers = {'user-agent': 'Mozilla/5.0'}
response = requests.get(url, headers=headers)
tree = html.fromstring(response.content)

# 使用 XPath 定位 id 为 'constituents' 的表格的所有行
table_rows = tree.xpath('//*[@id="constituents"]/tbody/tr')
print(f"找到 {len(table_rows)} 行数据")
```
</Example>

### 3. Selenium：处理动态内容

当网页数据是通过 JavaScript 动态加载时，BeautifulSoup 无法直接获取。这时需要 <Term id="selenium" en="Selenium">Selenium</Term>。它模拟真实浏览器，能执行 JavaScript，等待页面加载完成，并模拟用户操作（如点击、滚动）。

<KeyPoint>
**何时使用 Selenium？** 当你在网页源码中看不到目标数据，但浏览器渲染后能看到时，就需要 Selenium。
</KeyPoint>

### 4. yfinance：金融数据的快捷方式

对于金融数据，有更省力的办法。<Term id="yfinance" en="yfinance">yfinance</Term> 是一个非官方的 Python 库，专门用于从 Yahoo Finance 获取金融数据。它封装了复杂的请求和解析过程，几行代码就能拿到数据。

<Example title="使用 yfinance 获取股票历史数据">
```python
import yfinance as yf

# 获取腾讯控股 (0700.HK) 最近一个月的历史数据
ticker = yf.Ticker("0700.HK")
data = ticker.history(period="1mo")
print(data.head())
```
</Example>

### 实战案例：抓取 S&P 500 成分股

这是一个综合案例，展示如何从 Wikipedia 抓取 S&P 500 成分股列表。

<Example title="抓取 S&P 500 成分股">
```python
import requests
from bs4 import BeautifulSoup
import pandas as pd

headers = {
    "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/139.0.0.0 Safari/537.36"
}
url = 'https://en.wikipedia.org/wiki/List_of_S%26P_500_companies'
text = requests.get(url, headers=headers).text
soup = BeautifulSoup(text, 'html.parser')

# 通过 id 定位正确的表格
table = soup.find('table', id="constituents")

# 提取表头
headers_list = [th.text.strip() for th in table.find_all('th')]

# 提取数据行
rows = []
for tr in table.find_all('tr')[1:]:  # 跳过表头行
    cells = tr.find_all('td')
    row = [cell.text.strip() for cell in cells]
    rows.append(row)

# 创建 DataFrame
df = pd.DataFrame(rows, columns=headers_list)
print(df.head())
```
</Example>

## 网页抓取的局限性 {#scraping-limitations}

网页抓取虽然强大，但并非万能，存在以下主要局限性：

1.  **数据质量**：网站数据可能不完整、不及时，影响分析准确性。
2.  **资源密集型**：抓取大量数据会消耗大量带宽和存储空间，且加载页面耗时。
3.  **技术挑战**：
    - **网站结构变化**：网站改版可能导致基于特定标签或 class 的定位代码失效。
    - **反爬措施**：许多网站会实施 IP 封锁、请求频率限制、验证码等反爬措施。

## 调整收盘价：理解价格背后的故事 {#adjusted-close}

在金融数据中，你经常会看到两列价格：`Close`（收盘价）和 `Adj Close`（调整收盘价）。它们有什么区别？

- **收盘价 (Close)**：当天最后一笔交易的价格。
- **调整收盘价 (Adjusted Close)**：经过分红、拆股等公司事件调整后的收盘价。

<KeyPoint>
**为什么需要调整？** 公司事件（如分红、拆股）会导致股价出现非市场因素的跳变。例如，2:1 拆股后，股价会瞬间减半。如果使用原始收盘价进行长期回测，会错误地认为股价暴跌，导致策略表现被严重低估。调整收盘价消除了这些事件的影响，使历史价格序列连续，真实反映股票的价值变化。
</KeyPoint>

### 调整机制

Yahoo Finance 使用乘数法向后调整历史价格。

- **拆股调整**：对于 2:1 拆股，拆股前的所有价格乘以 `0.5`。
- **分红调整**：对于每股 $0.08 的分红，除息日前的所有价格乘以 `(1 - 0.08 / 除息日前一日收盘价)`。

<Example title="计算调整收盘价">
假设一只股票在 2/18 进行 2:1 拆股，2/21 除息（分红 $0.08）。2/17 收盘价为 $48.30，2/18 收盘价为 $24.96。

1.  **计算调整因子**：
    - 拆股因子 = 0.5
    - 分红因子 = 1 - (0.08 / 24.96) = 0.9968
2.  **计算 2/17 的调整收盘价**：
    - 调整收盘价 = 0.5 * 0.9968 * 48.30 = **24.07**
</Example>

## 使用 Python 管理 SQLite 数据库 {#sqlite-database}

抓取下来的数据不能每次都重新爬，你需要一个本地数据库。Python 内置了 <Term id="sqlite" en="SQLite">SQLite</Term>，一个文件就是一个数据库，零配置，非常适合学习和中小型项目。

### 核心操作 (CRUD)

<Definition title="CRUD">
指数据库的四种基本操作：**C**reate (创建/插入), **R**ead (查询), **U**pdate (更新), **D**elete (删除)。
</Definition>

<Example title="SQLite 基本操作">
```python
import sqlite3

# 1. 连接数据库 (如果不存在则创建)
conn = sqlite3.connect('market.db')
cursor = conn.cursor()

# 2. 创建表 (CREATE)
cursor.execute('''
    CREATE TABLE IF NOT EXISTS stocks (
        symbol TEXT,
        date TEXT,
        close_price REAL
    )
''')

# 3. 插入数据 (INSERT)
cursor.execute("INSERT INTO stocks (symbol, date, close_price) VALUES (?, ?, ?)", 
               ('0700.HK', '2025-01-02', 412.4))
conn.commit()

# 4. 查询数据 (SELECT)
cursor.execute("SELECT * FROM stocks WHERE symbol = ?", ('0700.HK',))
for row in cursor.fetchall():
    print(row)

# 5. 更新数据 (UPDATE)
cursor.execute("UPDATE stocks SET close_price = ? WHERE symbol = ? AND date = ?", 
               (415.0, '0700.HK', '2025-01-02'))
conn.commit()

# 6. 删除数据 (DELETE)
cursor.execute("DELETE FROM stocks WHERE symbol = ?", ('0700.HK',))
conn.commit()

# 7. 关闭连接
conn.close()
```
</Example>

## 金融数据存储的数据库设计 {#database-design}

设计存储金融数据的数据库时，没有完美的方案，关键在于权衡。主要考虑三种策略：

### 策略一：所有股票放在同一张大表

- **优点**：结构简单，易于跨股票查询和分析。
- **缺点**：表会非常庞大，查询和备份速度慢。

### 策略二：按股票代码分表

- **优点**：单只股票的回测查询非常快。
- **缺点**：表数量庞大，跨股票分析需要查询多张表，代码复杂。

### 策略三：按时间（如月份）分表

- **优点**：按时间范围查询快，便于月度数据管理。
- **缺点**：跨月查询需要动态拼接 SQL，单只股票的完整历史查询较慢。

<KeyPoint>
**如何选择？**
- **策略回测**：主要查询单只股票的完整历史，推荐**按股票分表**。
- **数据分析**：主要进行跨股票的截面分析，推荐**所有股票放在同一张大表**并建立合适的索引。
</KeyPoint>

## 易错点 {#pitfalls}

1.  **忽略 User-Agent**：直接使用 `requests.get(url)` 可能会被服务器拒绝。务必设置 `headers` 模拟浏览器访问。
2.  **混淆 `Close` 与 `Adj Close`**：进行长期回测时，必须使用 `Adj Close`，否则结果会严重失真。
3.  **数据库设计一刀切**：没有最好的设计，只有最适合当前场景的设计。不要盲目选择一种策略，要根据主要用途（回测 vs 分析）来权衡。
4.  **忘记提交事务**：在 SQLite 中执行 `INSERT`、`UPDATE`、`DELETE` 后，必须调用 `conn.commit()` 才能使更改生效。
