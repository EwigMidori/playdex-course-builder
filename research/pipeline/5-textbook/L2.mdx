# 数据抓取与数据库管理

## 为什么这一课重要 {#why-this-lesson-matters}

想象一下，你正在监控几百只股票的价格变化。手动刷新网页？那会累死人。你需要一个自动化的方式，让程序帮你从网站上把数据拿下来，然后存到一个地方，方便以后分析。

这就是本课的核心：**网页抓取**与**数据库管理**。它们是量化交易和金融分析的基础设施。没有数据，一切策略都是空谈。

<KeyPoint>
本课将带你掌握从网页获取数据、用 Python 库解析数据、理解金融数据中的关键概念（如调整收盘价），以及将数据存入数据库的完整流程。
</KeyPoint>

## 课程全景与关键问题 {#concept-map}

本课围绕以下六个核心问题展开：

1.  **网页由什么构成？** 了解 HTML、CSS、JavaScript 的角色，是精准抓取数据的前提。
2.  **如何用 Python 抓取网页数据？** 学习 BeautifulSoup、lxml/XPath、Selenium、yfinance 等工具。
3.  **网页抓取有什么局限？** 了解数据质量、资源消耗、反爬虫等技术挑战。
4.  **如何用 Python 创建和管理数据库？** 学习使用 SQLite 进行数据的增删改查。
5.  **什么是调整收盘价？** 理解这个金融数据中的核心概念，以及它为什么对回测至关重要。
6.  **如何设计金融数据库？** 探讨按股票分区 vs 按时间分区的权衡。

## 网页的基本结构 {#web-structure}

一个典型的网站，通常由三部分构成：

-   **<Term id="html" en="HyperText Markup Language">HTML</Term>**：定义网页的**结构**和**内容**。它用标签来组织标题、段落、表格等元素。HTML 是一种标记语言，不是编程语言。
-   **<Term id="css" en="Cascading Style Sheets">CSS</Term>**：控制网页的**视觉样式**。颜色、字体、间距，都靠它来美化。CSS 的规则是“层叠”的，多个样式可以应用于同一个元素，并有优先级规则决定哪个生效。
-   **<Term id="javascript" en="JavaScript">JavaScript</Term>**：让网页**动起来**。点击按钮、弹出窗口、动态加载内容，都是它的功劳。

<Definition title="HTML 标签">
HTML 使用成对的标签（如 `<h1>` 和 `</h1>`）来定义元素。常见的标签包括：
- `<h1>` 到 `<h6>`：标题
- `<p>`：段落
- `<a>`：超链接
- `<img>`：图片
- `<table>`、`<tr>`、`<td>`：表格
- `<div>`、`<span>`：容器
</Definition>

## Python 网页抓取工具 {#web-scraping-tools}

<Term id="web_scraping" en="Web Scraping">网页抓取</Term>是用程序自动从网页中提取数据的过程。Python 提供了多种强大的库来完成这项任务。

### 1. BeautifulSoup

<Term id="beautifulsoup" en="BeautifulSoup">BeautifulSoup</Term> 是一个流行的 Python 库，用于解析和搜索 HTML 与 XML 数据。

**基本工作流程：**

1.  使用 `requests` 库下载网页的原始 HTML 内容。
2.  将内容传递给 `BeautifulSoup` 进行解析，得到一个 `soup` 对象。
3.  使用 `soup.find()` 或 `soup.find_all()` 方法定位并提取所需元素。

<Example title="提取所有段落">
```python
import requests
from bs4 import BeautifulSoup

# 1. 下载网页内容
text = requests.get('https://example.com').text

# 2. 解析 HTML
soup = BeautifulSoup(text, 'html.parser')

# 3. 提取所有 <p> 标签
all_paragraphs = soup.find_all('p')
for p in all_paragraphs:
    print(p.text)
```
</Example>

<KeyPoint>
- `soup.find('p')`：返回第一个 `<p>` 标签。
- `soup.find_all('p')`：返回所有 `<p>` 标签的列表。
- `soup.get_text()`：提取网页中所有文本。
</KeyPoint>

### 2. lxml 与 XPath

<Term id="xpath" en="XPath">XPath</Term> 是一种在 XML/HTML 文档中导航和选择节点的查询语言。配合 <Term id="lxml" en="lxml">lxml</Term> 库，你可以用 XPath 表达式精确定位元素。

**常见 XPath 语法：**

-   `//element`：选择文档中所有名为 `element` 的节点。
-   `//element[@attribute='value']`：选择属性 `attribute` 值为 `value` 的 `element` 节点。
-   `//element[position()=1]`：选择第一个 `element` 节点。

<Example title="使用 XPath 定位表格">
```python
from lxml import html
import requests

# 下载网页
response = requests.get('https://example.com')
tree = html.fromstring(response.content)

# 使用 XPath 定位 id 为 'constituents' 的表格中的所有行
table_rows = tree.xpath('//*[@id="constituents"]/tbody/tr')
```
</Example>

### 3. Selenium

当目标网站的数据是通过 JavaScript 动态加载的，BeautifulSoup 和 lxml 就无法直接获取了。这时需要 <Term id="selenium" en="Selenium">Selenium</Term>。

Selenium 可以模拟真实用户的操作：打开浏览器、点击按钮、滚动页面。等页面加载完成后，再提取数据。

<KeyPoint>
**适用场景**：处理动态加载内容的网页。
**工作原理**：自动化浏览器操作，等待 JavaScript 执行完毕。
</KeyPoint>

### 4. yfinance

对于金融数据，有一个更省力的选择。<Term id="yfinance" en="yfinance">yfinance</Term> 是一个专门从 Yahoo Finance 获取数据的非官方 Python 库。

<Example title="获取股票历史数据">
```python
import yfinance as yf

# 获取腾讯控股 (0700.HK) 最近一个月的数据
ticker = yf.Ticker("0700.HK")
data = ticker.history(period="1mo")
print(data)
```
</Example>

## 网页抓取的局限性 {#scraping-limitations}

网页抓取虽然强大，但并非万能，存在以下主要局限性：

1.  **数据质量**：网站上的数据可能不完整、不准确或不是最新的。
2.  **资源密集**：抓取大量数据会消耗大量带宽和存储空间，加载网页内容也很耗时。
3.  **技术挑战**：
    -   网站结构可能频繁变化，导致抓取代码失效。
    -   许多网站会实施反爬虫措施，如 IP 封锁和速率限制。

<KeyPoint>
在开始抓取前，务必评估数据源的质量和稳定性，并尊重网站的 `robots.txt` 协议。
</KeyPoint>

## 使用 Python 管理 SQLite 数据库 {#database-management}

抓取数据只是第一步，你还需要一个地方来存储和管理这些数据。Python 内置了对 <Term id="sqlite" en="SQLite">SQLite</Term> 的支持，它是一个轻量级的、基于磁盘的数据库引擎。

### 基本 CRUD 操作

<Definition title="SQL CRUD 操作">
- **C**reate (创建): `INSERT`
- **R**ead (读取): `SELECT`
- **U**pdate (更新): `UPDATE`
- **D**elete (删除): `DELETE`
</Definition>

<Example title="创建表并插入数据">
```python
import sqlite3

# 1. 连接数据库（如果不存在则创建）
conn = sqlite3.connect('market.db')
cursor = conn.cursor()

# 2. 创建表
cursor.execute('''
    CREATE TABLE IF NOT EXISTS market_candles (
        symbol text,
        timestamp text,
        close_price float
    )
''')

# 3. 插入数据（使用参数化查询防止 SQL 注入）
cursor.execute('INSERT INTO market_candles VALUES (?, ?, ?)', 
               ('AAPL', '2025-01-02', 242.75))

# 4. 提交事务
conn.commit()

# 5. 查询数据
cursor.execute('SELECT * FROM market_candles')
rows = cursor.fetchall()
for row in rows:
    print(row)

# 6. 关闭连接
conn.close()
```
</Example>

<KeyPoint>
- 使用 `sqlite3.connect()` 连接数据库。
- 使用 `cursor.execute()` 执行 SQL 语句。
- 使用参数化查询（`?` 占位符）来安全地插入数据。
- 操作完成后记得 `conn.commit()` 和 `conn.close()`。
</KeyPoint>

## 调整收盘价的概念与计算 {#adjusted-close}

在存储金融数据时，有一个非常重要的概念：<Term id="adjusted_close" en="Adjusted Close">调整收盘价</Term>。

### 为什么需要调整？

原始收盘价会因为公司事件（如分红、拆股）出现不自然的跳变。例如，一只股票在 2:1 拆股后，价格会突然变成原来的一半，但这不代表公司价值缩水了。

调整收盘价就是把这些事件的影响“抹平”，它向后修正历史价格，让你看到更真实的长期走势。

### 调整因子

调整收盘价的计算基于两个乘数：

1.  **拆分乘数 (Split Multiplier)**：由拆股比例决定。
    -   2:1 拆股，乘数为 0.5。
    -   1:5 合股，乘数为 5。
2.  **股息乘数 (Dividend Multiplier)**：基于股息占除息前一日收盘价的百分比计算。
    -   公式：`1 - (股息 / 除息前一日收盘价)`

<Example title="调整收盘价计算示例">
假设一只股票在 2 月 18 日进行了 2:1 拆股，2 月 21 日是除息日（每股派发 0.08 元）。

| 日期 | 事件 | 收盘价 |
| :--- | :--- | :--- |
| 2/16 | - | 46.99 |
| 2/17 | - | 48.30 |
| 2/18 | 2:1 拆股 | 24.96 |
| 2/19 | - | 24.91 |
| 2/20 | - | 24.95 |
| 2/21 | 除息 0.08 | 24.53 |
| 2/22 | - | 24.54 |

**计算调整收盘价：**

-   **拆分乘数** = 0.5
-   **股息乘数** = 1 - (0.08 / 24.96) = 0.9968

| 日期 | 调整收盘价计算 | 调整收盘价 |
| :--- | :--- | :--- |
| 2/16 | 0.5 * 0.9968 * 46.99 | 23.42 |
| 2/17 | 0.5 * 0.9968 * 48.30 | 24.07 |
| 2/18 | 0.9968 * 24.96 | 24.88 |
| 2/19 | 0.9968 * 24.91 | 24.83 |
| 2/20 | 0.9968 * 24.95 | 24.87 |
| 2/21 | 24.53 (无需调整) | 24.53 |
| 2/22 | 24.54 (无需调整) | 24.54 |
</Example>

<KeyPoint>
调整收盘价是向后修正的。每当公司发生新的分红、拆股等事件时，所有历史调整收盘价都需要重新计算。
</KeyPoint>

## 金融数据存储的数据库设计 {#database-design}

设计数据库时，没有完美的方案，你需要根据使用场景来权衡。以下是两种常见的设计思路：

### 方案一：按股票分区

为每只股票创建一个单独的表。

-   **优点**：针对单只股票的查询非常快，数据备份也更容易。
-   **缺点**：进行跨股票分析时需要动态生成 SQL 语句，管理多个表比较麻烦。
-   **适用场景**：主要做回测（针对单只股票）。

### 方案二：按时间分区

将数据按月或按季度分成不同的表。

-   **优点**：可以快速查询某个时间段内所有股票的数据，适合跨资产分析。
-   **缺点**：如果要查询某只股票的完整历史，可能需要跨多个分区表进行查询。
-   **适用场景**：主要做跨资产分析（如计算相关性）。

<KeyPoint>
**没有银弹，只有权衡。** 关键是在存储空间、查询速度和内存之间找到平衡。
</KeyPoint>

## 易错点 {#pitfalls}

1.  **混淆 HTML、CSS、JavaScript 的职责**：记住，HTML 是结构，CSS 是样式，JavaScript 是行为。
2.  **忽略反爬虫机制**：直接使用 `requests.get()` 抓取某些网站（如 Yahoo Finance）可能会失败，因为网站会检查 `User-Agent`。需要设置合适的请求头，或使用专门的库（如 `yfinance`）。
3.  **混淆原始收盘价与调整收盘价**：在回测时，务必使用调整收盘价，否则结果会被公司事件扭曲。
4.  **认为调整收盘价是固定不变的**：调整收盘价会随着新的公司事件（如分红）而向后更新，因此数据库中的历史数据可能需要定期刷新。
5.  **认为存在完美的数据库设计方案**：数据库设计是权衡的艺术，没有“最好”的方案，只有“最适合当前场景”的方案。
