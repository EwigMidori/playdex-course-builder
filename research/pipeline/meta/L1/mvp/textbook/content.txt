# 算法交易基础与金融市场

## 为什么这一课重要 {#why-this-lesson-matters}

想象一下，你坐在电脑前，屏幕上跳出一个信号：“买入”。不到一秒钟，订单已经发出，交易自动完成。这不是科幻电影，这就是**算法交易**。

如今，算法交易已经占据了美国股市约60%的交易量，在欧洲是40%，在外汇市场是25%。预计到2028年，全球算法交易市场将以12.7%的年复合增长率增长至320亿美元。掌握算法交易的基础知识，是理解现代金融市场运作的关键一步。

本课将为你建立算法交易的知识框架，从基本概念到市场机制，从交易方式对比到基础设施需求，为你后续深入学习策略设计、回测、风险管理等内容打下坚实基础。

## 课程全景与关键问题 {#concept-map}

### 什么是算法交易？

<Definition title="算法交易 (Algorithmic Trading)">
算法交易，也称为量化交易、机器人交易或程序化交易，是指使用数学模型和计算机程序来生成交易信号（决策）并自动执行交易（执行）的方式。
</Definition>

算法交易的核心目标包括：
- **最大化利润**
- **控制执行成本**
- **对冲和管理投资风险**

<KeyPoint>
算法交易 = 数学模型 + 计算机程序 → 交易决策 + 自动执行
</KeyPoint>

<Example title="算法交易的实际场景">
假设你开发了一个策略：当某只股票的5日均线上穿20日均线时买入。在算法交易中，计算机会实时监控数据，一旦条件满足，自动生成买入信号并立即下单，整个过程无需人工干预。
</Example>

### 市场趋势

算法交易市场正在快速增长：
- **全球市场规模**：预计到2028年达到320亿美元（CAGR 12.7%）
- **区域分布**：北美是最大市场，亚太地区增长最快
- **市场渗透率**：
  - 美国股票交易量：约60%
  - 欧洲股票交易量：约40%
  - 外汇交易：约25%
  - 美国期权交易：约20%

### 三种交易方式对比

| 维度 | 人类交易 | 机器人投顾 | 机器人交易 |
|------|---------|-----------|-----------|
| 工作时间 | ~9×5 | ~9×5 | 24×7 |
| 执行速度 | 慢 | 中等 | 立即 |
| 数据输入 | 有限 | 几乎无限 | 几乎无限 |
| 交易频率 | 低 | 低-中 | 低-高 |
| 可扩展性 | 有限（受压力限制） | 中等 | 高 |
| 纪律性 | 不稳定（尤其亏损时） | 一致 | 高 |
| 定制化 | 高 | 中等 | 低 |
| 用户控制 | 完全 | 部分 | 最小 |
| 风险管理 | 主观 | 算法化 | 算法化 |

<KeyPoint>
机器人投顾（Robo-Advisory）提供建议但不执行交易，而机器人交易（Robo-Trading）则完全自动化执行。
</KeyPoint>

### 基础设施：数据是燃料，算法是引擎

部署算法交易程序需要考虑以下基础设施：

1. **数据采集**：数据来源和数据清洗耗时费力
2. **数据存储**：大量非结构化数据（数值、文本、图像、音频、视频）
3. **数据管理**：文件和数据格式不一致，传统数据库难以保护和管理
4. **数据处理**：可行性和效率
5. **其他因素**：
   - 实时数据源和历史数据
   - 计算硬件
   - 交易所/券商连接
   - 网络和延迟
   - 订单路由系统
   - 订单管理系统
   - 算法维护
   - 网络安全

## 核心概念与机制 {#core-concepts}

### 多头与空头

<Definition title="多头头寸 (Long Position)">
买入资产，预期价格上涨后卖出获利。例如：以$50买入100股，当价格上涨到$70时卖出获利。
</Definition>

<Definition title="空头头寸 (Short Position)">
卖出借来的资产，预期价格下跌后低价买回获利。做空的第一步是从券商借入股票。
</Definition>

<Example title="做空操作流程">
1. **借入股票**：从券商借入100股
2. **卖出**：以当前价格$70卖出
3. **买回**：当价格下跌到$50时买回100股
4. **归还**：将股票归还券商
</Example>

### 订单簿

<Definition title="订单簿 (Order Book)">
实时列出所有买入和卖出挂单的电子列表，按价格排序。它是市场的“记账本”。
</Definition>

订单簿的关键组成部分：
- **买单簿 (Bid Book)**：记录所有买入订单
- **卖单簿 (Ask Book)**：记录所有卖出订单
- **价格**：投资者愿意交易的价格点
- **数量**：投资者想买入或卖出的数量

订单簿示例：买单按价格从高到低排列，卖单按价格从低到高排列
</Figure>

### 买入价、卖出价与价差

<Definition title="买入价 (Bid Price)">
买方愿意支付的最高价格。例如，如果买方出价$207.01 × 3000，表示愿意以不超过$207.01的价格买入3000股。
</Definition>

<Definition title="卖出价 (Ask Price)">
卖方愿意接受的最低价格。例如，如果卖方要价$209.94 × 200，表示愿意以不低于$209.94的价格卖出200股。
</Definition>

<Definition title="买卖价差 (Bid-Ask Spread)">
卖出价与买入价之间的差额，是衡量市场流动性的关键指标。价差越小，流动性越好。
</Definition>

<Example title="价差计算">
某股票的买入价为$24，卖出价为$25，则买卖价差 = $25 - $24 = $1。
</Example>

<QuestionRef id="q_flash_bid_ask_spread_v1" />

### 订单类型

<Definition title="市价单 (Market Order)">
以当前市场上可获得的最佳价格立即买入或卖出的订单。保证成交，但不保证价格。
</Definition>

<Definition title="限价单 (Limit Order)">
指定一个价格（限价），只在该价格或更优价格成交的订单。保证价格，但不保证成交。
</Definition>

<Definition title="止损单 (Stop Order)">
当价格触及指定价格（止损价）时，自动变为市价单执行的订单。常用于控制亏损或捕捉突破行情。
</Definition>

<Example title="限价单的实际应用">
当前股价约为$138。如果你想在股价跌到$131.78时买入，可以设置一个买入限价单，限价为$131.78。当股价触及该价格时，订单会被执行。
</Example>

### 滑点

<Definition title="滑点 (Slippage)">
预期成交价格与实际成交价格之间的差异。
</Definition>

滑点通常发生在：
- 市场波动剧烈时
- 订单量太大，市场深度不足时

<Example title="滑点示例">
你的算法检测到股价跌到$50时触发买入信号，但由于市场波动，实际成交价为$51。这个$1的差异就是滑点。如果你计划买入100股，原本预期花费$5,000，实际花费$5,100，多出的$100就是滑点成本。
</Example>

### 杠杆与保证金

<Definition title="杠杆 (Leverage)">
使用借入资金放大投资回报（也放大风险）的方式。
</Definition>

<Definition title="保证金 (Margin)">
投资者为获得杠杆交易而必须存入的自有资金。
</Definition>

<KeyPoint>
杠杆是双刃剑：收益放大多少倍，亏损也放大多少倍。
</KeyPoint>

<Example title="杠杆的实际影响">
如果保证金要求是10%，那么杠杆就是10倍。用1万元本金，可以操作价值10万元的资产。如果资产价格上涨10%，你的收益率是100%；但如果价格下跌10%，你的亏损率也是100%。
</Example>

<QuestionRef id="q_flash_leverage_margin_v1" />

## 公式、图表与关键表征 {#formal-representation}

### 百分比价差公式

<Formula latex="\text{Percentage Spread} = \frac{\text{AskPrice} - \text{BidPrice}}{\text{MidPrice}} \times 100\% = \frac{2 \times (\text{AskPrice} - \text{BidPrice})}{\text{AskPrice} + \text{BidPrice}} \times 100\%">
百分比价差用于比较不同资产的流动性
</Formula>

<Example title="百分比价差计算">
某股票买入价为$24，卖出价为$25：
- 买卖价差 = $25 - $24 = $1
- 百分比价差 = 2 × (25 - 24) / (25 + 24) × 100% ≈ 4.08%
</Example>

### 杠杆比率公式

<Formula latex="\text{Leverage Ratio} = \frac{1}{\text{Margin Requirement}}">
杠杆比率等于1除以保证金要求
</Formula>

<Example title="杠杆比率计算">
如果保证金要求为25%：
- 杠杆比率 = 1 / 0.25 = 4倍
- 投入$10,000本金，可操作$40,000的资产
- 资产上涨10%：收益 = $40,000 × 10% = $4,000，收益率 = 40%
- 资产下跌10%：亏损 = $40,000 × 10% = $4,000，亏损率 = 40%
</Example>

## 易错点 {#pitfalls}

<Pitfall>
**做空的风险是无限的**：如果股价持续上涨，做空者的亏损理论上没有上限。这与做多不同，做多的最大亏损仅限于投入的本金。
</Pitfall>

<Pitfall>
**市价单 vs 限价单的混淆**：市价单保证成交但不保证价格，限价单保证价格但不保证成交。在波动剧烈的市场中，使用市价单可能产生较大的滑点。
</Pitfall>

<Pitfall>
**杠杆的放大效应**：杠杆不仅放大收益，也等比例放大亏损。许多新手只看到杠杆带来的高收益潜力，而忽视了同等比例的高风险。
</Pitfall>

<Pitfall>
**滑点不是交易成本**：滑点不是佣金或手续费，而是由于市场条件导致的预期价格与实际成交价之间的差异。在大额交易或波动市场中，滑点可能成为主要的隐性成本。
</Pitfall>
