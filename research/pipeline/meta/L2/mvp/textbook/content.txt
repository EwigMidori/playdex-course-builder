# 数据爬取与数据库管理

## 为什么这一课重要 {#why-this-lesson-matters}

想象一下，你需要每天手动收集几百只股票的价格、新闻标题、社交媒体情绪……那会花掉你多少时间？

在算法交易中，数据是一切策略的基础。没有数据，就无法回测、无法分析、无法决策。本课将教你如何用 Python 自动完成这些重复劳动——从网页中提取数据，并将它们存入本地数据库，为后续的策略开发打下坚实基础。

<KeyPoint>
本课的核心目标：掌握从网络获取金融数据并系统化存储的完整流程。
</KeyPoint>

## 课程全景与关键问题 {#concept-map}

### 什么是网络爬取？

<Term id="web_scraping" en="web scraping">网络爬取</Term> 是从网站自动提取数据的过程。在算法交易中，它的应用非常广泛：

- **实时数据收集**：经济指标、股票价格等
- **市场情绪分析**：新闻、论坛、博客、社交媒体
- **公司分析**：产品公告、年报、董事变更
- **另类数据**：天气、网页流量等

### 网页的基本结构

一个网页通常由三部分组成：

1. **HTML** (HyperText Markup Language)：定义网页的结构和内容
2. **CSS** (Cascading Style Sheets)：控制网页的视觉呈现和布局
3. **JavaScript**：为网页添加交互和动态行为

<Term id="html" en="HyperText Markup Language">HTML</Term> 不是编程语言，而是一种标记语言。它使用标签（tags）来创建各种元素，如标题、段落、链接、图片、表格等。

<Definition title="常见 HTML 标签">
| 标签 | 描述 |
|------|------|
| `<h1>` - `<h6>` | 定义标题，`<h1>` 最大，`<h6>` 最小 |
| `<p>` | 定义段落 |
| `<a>` | 创建超链接 |
| `<img>` | 插入图片 |
| `<table>` | 创建表格 |
| `<div>` | 定义区块容器 |
| `<ul>` / `<ol>` | 无序/有序列表 |
| `<li>` | 列表项 |
</Definition>

<Example title="HTML 基本结构示例">
```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>页面标题</title>
</head>
<body>
  <h1>一级标题</h1>
  <p>这是一个段落。</p>
</body>
</html>
```

</Example>

## 核心工具：BeautifulSoup、XPath、Selenium 与 yfinance {#web-scraping-tools}

### BeautifulSoup

<Term id="beautifulsoup" en="BeautifulSoup">BeautifulSoup</Term> 是 Python 中最流行的 HTML/XML 解析库之一。

<Definition title="BeautifulSoup 常用方法">
- `find(tag)`：返回第一个匹配的标签
- `find_all(tag)`：返回所有匹配标签的列表
- `get_text()`：提取网页中的所有纯文本
- `prettify()`：格式化打印 HTML 结构
</Definition>

<Example title="使用 BeautifulSoup 提取数据">
```python
import requests
from bs4 import BeautifulSoup

# 下载网页内容
text = requests.get('https://example.com').text

# 解析 HTML
soup = BeautifulSoup(text, 'html.parser')

# 提取第一个段落
first_p = soup.find('p')
print(first_p.text)

# 提取所有超链接
all_links = soup.find_all('a')
for link in all_links:
    print(link.text, link.get('href'))

# 提取所有纯文本
print(soup.get_text())
```
</Example>

<Example title="提取网页中的表格数据">
```python
import requests
import pandas as pd
from bs4 import BeautifulSoup

# 设置 User-Agent 以绕过反爬
headers = {
    "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
}

url = 'https://en.wikipedia.org/wiki/List_of_S%26P_500_companies'
text = requests.get(url, headers=headers).text
soup = BeautifulSoup(text, 'html.parser')

# 找到 id 为 'constituents' 的表格
table = soup.find('table', id="constituents")

# 提取表头
headers_list = [th.text.strip() for th in table.find_all('th')]

# 提取数据行
rows = []
for tr in table.find_all('tr')[1:]:  # 跳过表头行
    cells = tr.find_all('td')
    row = [cell.text.strip() for cell in cells]
    rows.append(row)

# 创建 DataFrame
df = pd.DataFrame(rows, columns=headers_list)
print(df)
```
</Example>

### XPath 与 lxml

<Term id="xpath" en="XML Path Language">XPath</Term> 是一种在 XML 文档中导航和选择节点的查询语言。

<Definition title="常见 XPath 语法">
- `//child`：选择文档中所有 child 元素
- `//element[@attribute='value']`：选择属性为特定值的元素
- `//element[position()=1]`：选择第一个元素
- `//li[2]`：选择第二个 `<li>` 元素
</Definition>

<Example title="使用 lxml 和 XPath 提取数据">
```python
from lxml import html
import requests
import pandas as pd

headers = {
    "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
}

url = 'https://en.wikipedia.org/wiki/List_of_S%26P_500_companies'
response = requests.get(url, headers=headers)
tree = html.fromstring(response.content)

# 使用 XPath 提取表格行
table_rows = tree.xpath('//*[@id="constituents"]/tbody/tr')

data = []
for row in table_rows[1:]:  # 跳过表头行
    cells = row.xpath('td')
    if len(cells) > 0:
        symbol = cells[0].text_content().strip()
        security = cells[1].text_content().strip()
        data.append([symbol, security])

df = pd.DataFrame(data, columns=['Symbol', 'Security'])
print(df)
```
</Example>

### Selenium：处理动态加载页面

<Term id="selenium" en="Selenium">Selenium</Term> 是一个 Web 自动化工具，可以模拟真实用户操作浏览器。

<Definition title="Selenium 的核心用途">
- 处理 JavaScript 动态加载的内容
- 模拟点击、滚动、输入等用户操作
- headless 模式：在后台运行浏览器，不显示界面，节省资源
</Definition>

<Example title="使用 Selenium 提取动态数据">
```python
from selenium import webdriver
from selenium.webdriver.chrome.service import Service
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from bs4 import BeautifulSoup
import pandas as pd

# 设置 WebDriver
service = Service('path/to/chromedriver')
driver = webdriver.Chrome(service=service)

# 访问页面
url = 'https://finance.yahoo.com/quote/0700.HK/history/'
driver.get(url)

# 等待表格加载
WebDriverWait(driver, 10).until(
    EC.presence_of_element_located((By.CSS_SELECTOR, 'table'))
)

# 获取页面源码并解析
page_source = driver.page_source
soup = BeautifulSoup(page_source, 'html.parser')

# 提取数据
table = soup.find('table')
# ... 后续提取逻辑

driver.quit()
```

### yfinance：专为金融数据而生

<Term id="yfinance" en="yfinance">yfinance</Term> 是 Yahoo Finance 的非官方 Python 接口，可以轻松下载金融数据。

<Definition title="yfinance 常用方法">
- `yf.Ticker("AAPL")`：创建股票对象
- `ticker.history(period='1mo')`：下载历史数据
- `ticker.info`：获取股票基本信息
- `yf.download("AAPL MSFT", start="2023-01-01")`：下载多只股票数据
- `ticker.actions`：获取分红和拆股信息
</Definition>

<Example title="使用 yfinance 下载股票数据">
```python
import yfinance as yf

# 下载单只股票
ticker = yf.Ticker("0700.HK")
data = ticker.history(period='1mo')
print(data)

# 下载多只股票
data_multi = yf.download("0700.HK 0005.HK AAPL", 
                         start="2025-01-01", 
                         end="2025-06-30")
print(data_multi)

# 获取股票基本信息
print(ticker.info)
```
</Example>

## 调整收盘价：概念与计算 {#adjusted-close}

### 为什么需要调整收盘价？

<Term id="adjusted_close" en="Adjusted Close">调整收盘价</Term> 是经过分红、拆股等公司事件调整后的收盘价，用于反映股票的真实历史表现。

<KeyPoint>
原始价格序列可能因公司事件而出现突然的跳跃或下跌。调整收盘价通过回溯调整历史价格，消除了这些事件的影响，使价格序列连续，适合长期回测和绩效分析。
</KeyPoint>

### 调整因素

1. **拆股（Stock Split）**：例如 2:1 拆股，拆股前的价格乘以 0.5
2. **分红（Dividend）**：例如每股分红 $0.08，拆股前价格乘以 (1 - 0.08/24.96) = 0.9968

<Example title="调整收盘价计算示例">
假设某股票在 2 月 18 日发生 2:1 拆股，2 月 21 日除息（每股分红 $0.08）。

| 日期 | 收盘价 | 事件 |
|------|--------|------|
| 2/16 | 46.99 | - |
| 2/17 | 48.30 | - |
| 2/18 | 24.96 | 2:1 拆股 |
| 2/19 | 24.91 | - |
| 2/20 | 24.95 | - |
| 2/21 | 24.53 | 除息 $0.08 |
| 2/22 | 24.54 | - |

**调整收盘价计算：**

- 拆股乘数 = 0.5
- 分红乘数 = 1 - 0.08/24.96 = 0.9968

| 日期 | 调整收盘价计算 | 调整收盘价 |
|------|----------------|------------|
| 2/16 | 0.5 × 0.9968 × 46.99 | 23.42 |
| 2/17 | 0.5 × 0.9968 × 48.30 | 24.07 |
| 2/18 | 0.9968 × 24.96 | 24.88 |
| 2/19 | 0.9968 × 24.91 | 24.83 |
| 2/20 | 0.9968 × 24.95 | 24.87 |
| 2/21 | 24.53（无需调整） | 24.53 |
| 2/22 | 24.54（无需调整） | 24.54 |
</Example>

## 数据库管理：SQLite 与金融数据存储 {#database-management}

### 为什么需要数据库？

<KeyPoint>
在算法交易中，使用数据库存储市场数据可以避免每次回测都重新爬取，保证数据的一致性和完整性，并支持复杂的 SQL 查询。
</KeyPoint>

### SQLite 基础

<Term id="sqlite" en="SQLite">SQLite</Term> 是一个轻量级的、基于磁盘的关系型数据库引擎，Python 内置支持。

<Definition title="SQLite 基本操作">
- **连接数据库**：`sqlite3.connect('example.db')`
- **创建表**：`CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT, age INTEGER)`
- **插入数据**：`INSERT INTO users (name, age) VALUES ('Alice', 30)`
- **查询数据**：`SELECT * FROM users WHERE age > 25`
- **更新数据**：`UPDATE users SET age=31 WHERE name='Alice'`
- **删除数据**：`DELETE FROM users WHERE name='Alice'`
</Definition>

<Example title="使用 SQLite 存储市场数据">
```python
import sqlite3
import yfinance as yf

# 连接数据库
conn = sqlite3.connect('market_data.db')
cursor = conn.cursor()

# 创建表
cursor.execute('''
    CREATE TABLE market_candles (
        symbol text,
        timestamp text,
        open_price float,
        high_price float,
        low_price float,
        close_price float,
        volume float
    )
''')

# 下载数据
symbol = "0700.HK"
ticker = yf.Ticker(symbol)
data = ticker.history(period='max')

# 插入数据
for index, row in data.iterrows():
    cursor.execute('''
        INSERT INTO market_candles VALUES (?, ?, ?, ?, ?, ?, ?)
    ''', (symbol, index, row['Open'], row['High'], 
          row['Low'], row['Close'], row['Volume']))

conn.commit()

# 查询数据
cursor.execute('''
    SELECT * FROM market_candles 
    WHERE symbol='0700.HK' 
    AND timestamp >= '2025-06-01' 
    AND timestamp <= '2025-06-30'
    ORDER BY timestamp
''')
for row in cursor.fetchall():
    print(row)

conn.close()
```
</Example>

### 金融数据库设计权衡

<Definition title="两种常见分区策略">
**按股票分区：**
- 优点：回测单只股票时查询速度快
- 缺点：跨资产分析需要跨多个表，实现复杂

**按日期分区：**
- 优点：跨资产分析方便，可以快速获取所有股票在某一时间段的数据
- 缺点：回测单只股票需要扫描多个分区，速度慢
</Definition>

<KeyPoint>
没有完美的设计方案！需要在存储空间、查询速度和内存占用之间找到平衡，根据实际使用场景选择最合适的方案。
</KeyPoint>

## 易错点 {#pitfalls}

<Pitfall title="忽略 User-Agent 导致请求被拒">
许多网站会检查请求的 User-Agent 头。如果不设置合适的 User-Agent，请求可能会被拒绝或返回错误页面。
```python
# 正确做法：设置 User-Agent
headers = {"user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"}
response = requests.get(url, headers=headers)
```
</Pitfall>

<Pitfall title="混淆原始收盘价与调整收盘价">
在回测中，如果使用原始收盘价而不是调整收盘价，分红和拆股事件会造成价格序列的虚假跳跃，导致回测结果不准确。
</Pitfall>

<Pitfall title="忽略数据库设计权衡">
将所有股票数据放在一张大表中会导致查询速度极慢。需要根据使用场景（回测 vs 数据分析）选择合适的分区策略。
</Pitfall>

<Pitfall title="忘记提交事务">
在使用 SQLite 时，执行 INSERT、UPDATE、DELETE 操作后必须调用 `conn.commit()`，否则更改不会保存到数据库。
</Pitfall>

<Pitfall title="忽略反爬措施">
频繁请求同一网站可能导致 IP 被封。应合理设置请求间隔，遵守网站的 robots.txt 协议。
</Pitfall>

### 关键要点回顾

<KeyPoint>
1. **网络爬取**：用程序自动从网站提取数据，在算法交易中用于收集股价、新闻、情绪等数据
2. **HTML 结构**：网页的骨架，爬虫通过解析 HTML 标签提取内容
3. **BeautifulSoup**：最常用的 HTML 解析库，提供 `find()`、`find_all()`、`get_text()` 等方法
4. **XPath**：强大的节点选择语言，配合 lxml 库解析速度更快
5. **Selenium**：处理动态加载页面的利器，可以模拟浏览器操作
6. **yfinance**：专为金融数据设计的库，一行代码即可下载股票历史数据
7. **调整收盘价**：消除分红、拆股等事件对价格序列的影响，是回测的基础
8. **SQLite**：轻量级数据库，Python 内置支持，适合存储金融数据
9. **数据库设计**：需要在存储空间、查询速度和内存占用之间权衡，根据场景选择分区策略
10. **爬取局限性**：数据质量、资源消耗、反爬措施是主要挑战
</KeyPoint>
